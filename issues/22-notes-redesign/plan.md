# Plan: Notes Page Redesign — Strip Filters & Folders, Keep Multi-Select

## Goal

Simplify the Notes tab to match the new Paper design and strip out the filter/folder/trash-visible UI. Keep all metadata in the schema — we just stop *showing* it.

## What changes

### Remove / unhook (but keep schema + backend intact)

- **Filter pills** (All / Manual / AI) — gone.
- **Filter drawer** (sort + show-filter: favourites / archived / trash) — gone.
- **Trash is dead as a concept.** Delete = **permanent** (`permanentlyDeleteNote`). No soft-delete path, no trash view, no untrash, no empty-trash confirm.
- **Favourites multi-select action** — gone. Action bar now has only Export + Delete.
- **Archive swipe action** — gone.
- **Folders / Collections** — the grid toggle button in the header that routes to `/collections` is gone.
- **FAB is gone entirely** — no "Add note", no "Ask AI", no "Create folder". Notes are passive: either auto-generated by the agent, or opened to read/edit. No in-app creation.
- **`From: …` conversation-title label** on AI notes — gone (conversations UI is deprecated).
- **The list/grid toggle** next to the filter button — gone entirely (list-only).
- **Swipe actions on rows** — gone. Deletion happens exclusively through multi-select + Delete.

Keep in the schema:
- `isFavourite`, `isArchived`, `isTrashed`, `folderId`, `isAIGenerated` fields all stay on the `Note` model. Manager methods (`favouriteNote`, `archiveNote`, `trashNote`, `untrashNote`, `permanentlyDeleteNote`, `emptyNoteTrash`) stay callable. We just don't wire UI to them right now — except `permanentlyDeleteNote`, which powers the new Delete action.

### Keep

- **Hold-to-select** multi-select flow (same `useMultiSelect` hook, `SelectionHeader`, `MultiSelectBar`, animations that match TranscriptList).
- **Two multi-select actions**: Export and Delete.
  - Export → `ExportDrawer` → Clipboard or Email.
  - Delete → batch `permanentlyDeleteNote` (permanent; gone from Mongo). Confirm drawer before delete since it's irreversible — "Delete N notes? This cannot be undone."
- **Tap to open a note** → `/note/{id}` (unchanged).

### New UI structure (from Paper spec)

- Warm stone bg `#FCFBFA`.
- Header: `MENTRA NOTES` red eyebrow + giant black "Notes" title (matches new HomePage / SearchPage style — `text-[34px]` black, tracking -0.5px). No action buttons in the header row.
- Content:
  - Section header per day. Labels: `Today`, `Yesterday`, else a short numeric date like `Mar 12` or `Mar 12, 2025` — no weekday names. Styled as plain dark label (`text-black text-base`) with `py-2 px-6`.
  - **Group by `createdAt` (timestamp) in the user's timezone, not `note.date`.** A note created at 11:58 PM on Wednesday stays in Wednesday's group even if the transcript `date` field says otherwise.
  - Each note row under it:
    - `border-t-[#F0EDEA]` between rows,
    - `border-l-[3px] border-l-[#D32F2F]` (red left rail on every note),
    - Title: `font-bold text-[17px] tracking-[-0.2px]`, color `#1A1A1A`,
    - Preview: `text-[13px] text-[#6B655D]`, 1-2 lines,
    - Meta: `Today, 2:10 PM` / `Yesterday, 2:10 PM` / `Mar 12, 2:10 PM` in `text-[11px] text-[#B0AAA2] font-medium` — always show absolute time; no weekday; no type badge (`AI`/`Manual`), no `From: X` label, no star.
    - Trailing chevron `#C5C0B8`.
- Rows sorted newest-first within each day; days sorted newest-first.
- Empty state (when there are zero notes): keep existing dot-art halftone SVG + simplified copy "No notes yet — your notes will appear here." No CTAs (FAB is gone). Since trash and filters are gone, this is the *only* empty state; there's no "nothing matches filters" variant anymore.

### Multi-select behaviour (unchanged except action set)

- Long-press a row → enter selection mode.
- Checkbox slot animates in on the row's left.
- Header swaps to `SelectionHeader` (Cancel / N selected / Select All).
- Bottom bar swaps in `MultiSelectBar` with **two** actions:
  - Export (`ExportIcon`) → opens `ExportDrawer` → Clipboard / Email.
  - Delete (`DeleteIcon`, danger variant) → confirm drawer ("Delete N notes? This cannot be undone."), then batch-call `permanentlyDeleteNote` and exit selection. **Permanent** — the rows are removed from Mongo, not just flagged trashed.
- Cancel / empty selection → exit selection mode.
- Same tab-bar slide-out behavior we wired for TranscriptList (via `useTabBar().setHidden`).

### Export drawer tweaks

- `ExportDrawer` currently shows a "Linked Conversation" toggle + sub-transcript toggle when `itemType="note"`. Since conversations are dormant, **hide both toggles for notes**. Drawer becomes: title + always-on "Note Content" + destination cards.
- The batch export handler in NotesPage drops all `includeLinkedNote`/`includeTranscript` branches (they'd always be false anyway). Keep the clipboard and email paths clean — just title, date, type, content.

### FAB simplification

- `NotesFABMenu` today has: Add Note / Ask AI / Create Folder. Drop the folder option.
- "Ask AI" routes to `/` (home) currently — keep as-is.

## Files to touch

**Frontend**
- `src/frontend/pages/notes/NotesPage.tsx` — rewrite. Remove filter state/drawer/pills, trash view, Favourites/archive actions, folders toggle, conversation-title linking, AI/Manual badges, FAB rendering. Group by `createdAt`-date and render the new Paper layout.
- `src/frontend/pages/notes/NoteRow.tsx` — restyle to new row design (red left rail, bigger title, simpler meta row, no badges, no star, no swipe actions). Keep `isSelecting` / `isSelected` / `longPressHandlers` props. Remove swipe-to-archive/trash entirely.
- `src/frontend/pages/notes/NotesFABMenu.tsx` — no longer rendered from NotesPage. Leave the file in place so other callers (if any) still work, but Notes no longer imports it.
- `src/frontend/components/shared/ExportDrawer.tsx` — for `itemType="note"`, hide the Linked Conversation + Transcript toggles.

**Backend** — no changes. All metadata stays.

## Followups / out of scope

- Bringing back filters, favourites UI, archive, trash view: deferred. When needed, just re-surface existing backend APIs.
- Folders / collections routing: deferred. Keep the pages in code; just don't link to them from Notes.
- Deep-link from Search results → tapping a note result still goes to `/note/{id}`; that already works.

## Acceptance

- No filter pills, no filter drawer button, no list/grid toggle, no trash/archive UI.
- Header = `MENTRA NOTES` eyebrow + giant "Notes" title. No buttons in the header.
- **No FAB** — no create-note or ask-AI affordance on this page.
- Notes grouped by **createdAt** day with labels `Today`, `Yesterday`, or `Mar 12` / `Mar 12, 2024` (no weekday names). Newest-first within each day and across days.
- Row shows title, 1-2 line preview, meta `(Today|Yesterday|Mar 12), 2:10 PM`, red 3px left rail, chevron. No type badges, no star, no "From: …", no swipe actions.
- Long-press → selection mode; bottom bar has exactly Export + Delete.
- **Delete is permanent**: confirm drawer, then `permanentlyDeleteNote` on each selected id.
- Export drawer for notes: only the content toggle + destination cards (Linked Conversation / Transcript toggles hidden for notes).
- Empty state uses existing dot-art halftone SVG; copy "No notes yet — your notes will appear here." No action cards/CTAs.
- `isFavourite`, `isArchived`, `isTrashed`, `folderId`, `isAIGenerated` all still exist in the schema; backend manager methods untouched.
